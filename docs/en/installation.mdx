This guide outlines the installation process for Alauda Cache Service for Redis OSS, including its required dependencies. Follow these steps sequentially to ensure proper deployment.

## Installing Alauda Container Platform Data Services Essentials

::: tip Optional
Alauda Container Platform Data Services Essentials provides the Web Console management interface components. This installation can be skipped for environments using CLI-only administration.
:::

### Prerequisites

1. Obtain the Alauda Container Platform Data Services Essentials installation package compatible with your platform version.
2. Use the platform's application publishing capability to publish Alauda Container Platform Data Services Essentials to the `global` cluster.

### Procedure

<Tabs>
  <Tab label="Web Console">
    1. Navigate to **Marketplace** > **Cluster Plugins** in the platform's left navigation panel.
    2. Select the `global` cluster from the cluster selector.
    3. Locate **Alauda Container Platform Data Services Essentials** in the plugin list and click **Install**.
  </Tab>
</Tabs>

## Installing Alauda Container Platform Data Services RDS Framework

::: tip Optional
Alauda Container Platform Data Services RDS Framework provides the api service management layer. This installation can be skipped for environments using CLI-only administration.
:::

### Prerequisites

1. Obtain the Alauda Container Platform Data Services RDS Framework installation package compatible with your platform version.
2. Use the platform's application publishing capability to publish Alauda Container Platform Data Services RDS Framework to any cluster where Data Services view capabilities are desired.

### Procedure

<Tabs>
  <Tab label="Web Console">
    1. Access the **Platform Management** interface.
    2. Navigate to **Marketplace** > **OperatorHub** in the left navigation panel.
    3. Locate **Alauda Container Platform Data Services RDS Framework** and click **Install** to access the deployment configuration page.

    Configure the installation parameters as follows:

    | Parameter | Recommended Configuration |
    | :-------- | :------------------------ |
    | **Channel** | **stable** - The production-ready distribution channel |
    | **Version** | Latest available version (selected by default) |
    | **Installation Mode** | **Cluster** - Provides a single Operator instance that manages Redis resources across all namespaces, optimizing resource utilization |
    | **Installation Location** | **Recommended Namespace** - The system will automatically create the appropriate namespace if it doesn't exist |
    | **Upgrade Strategy** | **Manual** - Operator upgrades require explicit approval, providing controlled update management |

    4. Review the configuration settings in the installation dialog and click **Install** to deploy the RDS Framework Operator.
  </Tab>
</Tabs>

## Installing Alauda Cache Service for Redis OSS

### Prerequisites

1. Obtain the installation package compatible with your platform version.
2. Upload the plugin to any cluster that intends to utilize the business component capabilities of Alauda Cache Service for Redis OSS.

### Procedure

<Tabs>
  <Tab label="Web Console">
    1. Access the **Platform Management** interface.
    2. Navigate to **Marketplace** > **OperatorHub** in the left navigation panel.
    3. Locate **Alauda Cache Service for Redis OSS** and click **Install** to access the deployment configuration page.

    Configure the installation parameters as follows:

    | Parameter | Recommended Configuration |
    | :-------- | :------------------------ |
    | **Channel** | The appropriate channel is automatically selected based on your platform version |
    | **Version** | Latest available version (selected by default) |
    | **Installation Mode** | **Cluster** - Provides a single Operator instance that manages Redis resources across all namespaces, optimizing resource utilization |
    | **Installation Location** | **Recommended Namespace** - The system will automatically create the appropriate namespace if it doesn't exist |
    | **Upgrade Strategy** | **Manual** - Operator upgrades require explicit approval, providing controlled update management |

    4. Review the configuration settings in the installation dialog and click **Install** to deploy Alauda Cache Service for Redis OSS.
  </Tab>
</Tabs>
