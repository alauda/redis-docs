# Backup and Restore

To safeguard against data loss, timely backups are highly recommended. In the event of a database anomaly, you can restore your data from a backup.

Alauda Cache Service for Redis OSS supports two backup types: S3 (Amazon Simple Storage Service) backups and PVC (PersistentVolumeClaim) backups.

* **S3 Backup:** Backs up data to S3 storage.
* **PVC Backup:** Backs up data to a PersistentVolume (PV) using a PersistentVolumeClaim (PVC). This method requires the PV to support Kubernetes node migration and the ReadWriteMany access mode. Examples of suitable PV types include NFS and Ceph.

## Prerequisites

To back up data to your own S3 bucket, please confirm with your administrator that S3 storage is properly configured for your project. Otherwise, ensure your business cluster provides a `StorageClass` that can provision network-based storage.


## Immediate Backup

Perform an immediate backup of the running Redis instance.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. In the left navigation panel, click **Redis**.

    2. Click on ***Namespace Name***.

    3. Click on ***Instance Name***.

    4. In the **Backup & Restore** tab, click **Create Backup**.

    5. Set the parameters and click **Create**. The duration of the backup depends on the data volume and network conditions, please be patient.
  </Tab>
</Tabs>

## Automatic Backup

Specify the backup time, after which the platform will automatically back up the Redis instance and perform backups regularly.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. In the left navigation panel, click **Redis**.

    2. Click on ***Namespace Name***.

    3. Click on ***Instance Name***.

    4. In the **Backup & Restore** tab, click the edit button in the **Automatic Backup Setting** area ![](../assets/44buttoneditblue.png).

    5. Enable **Automatic Backup** and set the parameters.

    6. Click **Update**.
  </Tab>
</Tabs>

## Restore

Restore the backup data to a new Redis instance.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. In the left navigation panel, click **Redis**.

    2. Click on ***Namespace Name***.

    3. Go to the source instance's **Backup & Restore** tab, and click ![](../assets/44buttonanniu.png) > **Recover**.

    4. Configure the instance parameters and click **Recover**. Please be patient until the target instance is in **Running** state.
  </Tab>
</Tabs>
