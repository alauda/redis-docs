# Patch Version Upgrade

With the need for vulnerability fixes or new feature releases, patch version upgrades of middleware will occur. During the patch version upgrade process, Pods will be restarted one by one, which may cause a brief interruption of service. Therefore, it is recommended to choose an appropriate time window for the upgrade. When a patch version is available for upgrade, it is advisable to upgrade as soon as possible to apply the latest fixes to the components.

## Setup Patch Version Upgrade Strategy

### Steps

<Tabs>
  <Tab label="Web Console">
    1. Go to the instance detail page.

    2. Click on **Patch Version Upgrade Policy**.

    3. Choose **Manual** or **Automatic**.

    4. Click on **Update** to complete the settings.
  </Tab>

</Tabs>

## Upgrade one by one

### Prerequisites

When a patch version is available, the instance list page will display an upgrade icon, and the instance detail page will show an upgrade prompt.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. Go to the instance detail page.

    2. Click **Upgrade Now** next to the upgrade prompt.

    3. Read the upgrade prompt to understand the version changes of components and plugins.

    4. Click **Upgrade Now**.

    5. Wait for the instance status to change to Running, indicating the upgrade was successful.
  </Tab>
</Tabs>

## Bulk Upgrade

### Prerequisites

Users must have platform administrator rights.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. Click on **Upgrade Management** in the left navigation bar.

    2. Select the **operator** you need to operate on, and go to the operator detail page.

    3. Select the instances you want to upgrade and click **Bulk Upgrade**.

    4. Read the upgrade prompt to understand the version changes of components and plugins.

    5. Click **Upgrade Now**.

    6. Wait for the instance status to change to Running, indicating the upgrade was successful.
  </Tab>

</Tabs>
