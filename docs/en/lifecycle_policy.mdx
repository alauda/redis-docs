# Lifecycle Policy

## Version Lifecycle Timeline

Below is the lifecycle schedule for released versions of the Alauda Cache Service for Redis OSS:

| Version | Release Date | End of Support |
|:--------|:-------------|:---------------|
| v4.1.x  | 2025-08-22   | 2027-08-30     |

## Release Policy

A new patch version of Alauda Cache Service for Redis OSS is released approximately every two to six months. Releases are driven by emerging security requirements.

A new minor version of Alauda Cache Service for Redis OSS is released approximately every six months. Releases are driven by significant updates to the underlying Redis server, the implementation of new features, or emerging security requirements.

## Maintenance Policy

During a version's maintenance period, Alauda provides the following services:
* Upstream Patching: We actively track official Redis patch releases and promptly deliver critical bug fixes and security vulnerability patches.
* Scheduled Security Updates: We release regular security updates for the service operator aligned with our two to six month patch release schedule to address vulnerabilities and ensure stability.
* Upgrade Assistance: We provide dedicated support to help customers execute smooth and seamless upgrades to newer versions.
