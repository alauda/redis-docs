## v4.0.0

### Новые и оптимизированные функции

#### Разделение компонентов и платформы

Срок службы Alauda Cache Services для Redis OSS теперь отделен от платформы, что означает, что когда клиенту нужна определенная функция или исправление ошибки, команда Redis может сразу выпустить обновление, не дожидаясь совместного релиза платформы.

#### Входящие логи на странице деталей компонента

Логи на странице деталей Redis были заменены на логи в реальном времени, которые обновляются мгновенно, что облегчает решение проблем и анализ неполадок.

#### Внешний доступ через LoadBalancer для Redis

Теперь поддерживается доступ к Redis через LoadBalancer. При создании экземпляра пользователи могут выбрать, чтобы предоставить доступ к услугам извне через сервис типа LoadBalancer.

В сравнении с NodePort, LoadBalancer обеспечивает более высокую доступность, производительность и экспорт стандартного порта. Рекомендуется использовать метод LoadBalancer в производственных средах.

##### Ограничения

При использовании LoadBalancer убедитесь, что в кластере есть доступные пула внешних адресов. Мы рекомендуем использовать MetalLB для создания внешних пулов адресов для кластера.
