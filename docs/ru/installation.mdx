Данное руководство описывает процесс установки службы кэширования Alauda для Redis OSS, включая необходимые зависимости. Следуйте этим шагам последовательно, чтобы обеспечить правильное развертывание.

## Установка основных компонентов служб данных Alauda Container Platform

::: tip Дополнительно
Основные компоненты служб данных Alauda Container Platform предоставляют компоненты управленческого интерфейса веб-консоли. Установка может быть пропущена для сред, использующих только интерфейс командной строки (CLI).
:::

### Предварительные требования

1. Получите пакет установки основных компонентов служб данных Alauda Container Platform, совместимый с вашей версией платформы.
2. Используйте возможности публикации приложений платформы для публикации основных компонентов служб данных Alauda Container Platform в кластер `global`.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Перейдите в **Marketplace** > **Cluster Plugins** в левой панели навигации платформы.
    2. Выберите кластер `global` в селекторе кластеров.
    3. Найдите **Основные компоненты служб данных Alauda Container Platform** в списке плагинов и нажмите **Установить**.
  </Tab>
</Tabs>

## Установка фреймворка RDS служб данных Alauda Container Platform

::: tip Дополнительно
Фреймворк RDS служб данных Alauda Container Platform предоставляет уровень управления сервисами API. Установка может быть пропущена для сред, использующих только интерфейс командной строки (CLI).
:::

### Предварительные требования

1. Получите пакет установки фреймворка RDS служб данных Alauda Container Platform, совместимый с вашей версией платформы.
2. Используйте возможности публикации приложений платформы для публикации фреймворка RDS служб данных Alauda Container Platform в любой кластер, где требуются возможности просмотра служб данных.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Перейдите в интерфейс **Администратора**.
    2. Перейдите в **Marketplace** > **OperatorHub** в левой панели навигации.
    3. Найдите **Фреймворк RDS служб данных Alauda Container Platform** и нажмите **Установить**, чтобы получить доступ к странице конфигурации развертывания.

    Настройте параметры установки следующим образом:

    | Параметр                 | Рекомендуемая конфигурация                                                                                                           |
    | :----------------------- | :------------------------------------------------------------------------------------------------------------------------------------ |
    | **Канал**                | **stable** - Канал распределения, готовый к производству                                                                                 |
    | **Версия**               | Последняя доступная версия (выбрана по умолчанию)                                                                                    |
    | **Режим установки**       | **Кластер** - Обеспечивает единственный экземпляр оператора, который управляет ресурсами Redis во всех пространствах имён, оптимизируя использование ресурсов |
    | **Место установки**       | **Рекомендуемое пространство имён** - Система автоматически создаст соответствующее пространство имён, если оно не существует                        |
    | **Стратегия обновления**  | **Ручная** - Обновления оператора требуют явного одобрения, что обеспечивает управляемое обновление                                     |

    4. Проверьте настройки конфигурации в диалоговом окне установки и нажмите **Установить** для развертывания оператора фреймворка RDS.
  </Tab>
</Tabs>

## Установка службы кэширования Alauda для Redis OSS

### Предварительные требования

1. Получите пакет установки, совместимый с вашей версией платформы.
2. Загрузите плагин в любой кластер, который намерен использовать возможности бизнес-компонентов службы кэширования Alauda для Redis OSS.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Перейдите в интерфейс **Администратора**.
    2. Перейдите в **Marketplace** > **OperatorHub** в левой панели навигации.
    3. Найдите **Служба кэширования Alauda для Redis OSS** и нажмите **Установить**, чтобы получить доступ к странице конфигурации развертывания.

    Настройте параметры установки следующим образом:

    | Параметр                 | Рекомендуемая конфигурация                                                                                                           |
    | :----------------------- | :------------------------------------------------------------------------------------------------------------------------------------ |
    | **Канал**                | Соответствующий канал автоматически выбирается на основе вашей версии платформы                                                      |
    | **Версия**               | Последняя доступная версия (выбрана по умолчанию)                                                                                    |
    | **Режим установки**       | **Кластер** - Обеспечивает единственный экземпляр оператора, который управляет ресурсами Redis во всех пространствах имён, оптимизируя использование ресурсов |
    | **Место установки**       | **Рекомендуемое пространство имён** - Система автоматически создаст соответствующее пространство имён, если оно не существует                        |
    | **Стратегия обновления**  | **Ручная** - Обновления оператора требуют явного одобрения, что обеспечивает управляемое обновление                                     |

    4. Проверьте настройки конфигурации в диалоговом окне установки и нажмите **Установить** для развертывания службы кэширования Alauda для Redis OSS.
  </Tab>
</Tabs>
