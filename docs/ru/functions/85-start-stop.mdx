# Запуск и остановка экземпляра

Служба кеша Alauda для Redis OSS предоставляет возможность однократного запуска и остановки, чтобы поддерживать оперативную гибкость в управлении экземплярами промежуточного ПО. Эта возможность особенно ценна для временного освобождения ресурсов без удаления экземпляров во время периодов обслуживания, при высокой нагрузке на узлы или в средах разработки/тестирования. Эффективное управление состоянием экземпляров помогает оптимизировать использование ресурсов и снижать операционные затраты.

## Остановка экземпляра

Остановка экземпляра Redis при этом аккуратно приостанавливает его работу, сохраняя конфигурацию и постоянные данные.

### Предварительные условия

Экземпляр должен находиться в одном из следующих состояний: **Обработка**, **Запуск**, **Аномальный** или **Неизвестный**.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    > **Рекомендация**: Выполните полное резервное копирование перед остановкой экземпляра, чтобы защитить себя от возможной потери данных.

    1. В левом навигационном меню нажмите **Redis**.

    2. Нажмите на ***название пространства имен***.

    3. Нажмите на ***название экземпляра***.

    4. Нажмите **Действия** > **Остановка** и подтвердите операцию.
  </Tab>
</Tabs>

### Функции, недоступные в состоянии остановки

| Функция                     | Ограничение                                                                                        |
| --------------------------- | ------------------------------------------------------------------------------------------------- |
| **Терминальная консоль**    | Доступ к экземплярам Redis через терминал недоступен.                                            |
| **Действия**                 | Операции с экземплярами, включая перезапуск, обновление и изменения конфигурации, отключены.    |
| **Топология**               | Визуализация топологии недоступна, поскольку компоненты экземпляра не работают.                  |
| **Конфигурация параметров** | Обновление параметров и экспорт конфигурации нельзя выполнить.                                   |
| **Управление пользователями**| Функциональность администрирования пользователей недоступна.                                     |
| **Резервное копирование и восстановление** | Новые операции резервного копирования не могут быть инициированы.                        |
| **Инспекция**               | Метрики и результаты инспекции для остановленного экземпляра недействительны.                    |
| **Способ доступа**          | Информация о маршрутизации может быть недоступна или недоступна.                                 |

## Запуск экземпляра

Запуск ранее остановленного экземпляра Redis восстанавливает полную функциональность и доступность сервиса.

### Предварительные условия

Экземпляр должен находиться в состоянии **Остановка** или **Остановлен**.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. В левом навигационном меню нажмите **Redis**.

    2. Нажмите на ***название пространства имен***.

    3. Нажмите на ***название экземпляра***.

    4. Нажмите **Действия** > **Запуск** и подтвердите операцию.
  </Tab>
</Tabs>

После успешного завершения операции запуска экземпляр переходит в состояние **Запуск**, и все функции становятся снова доступными.
