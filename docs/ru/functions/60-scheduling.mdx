# Конфигурация расписания

Сервис кэширования Alauda для Redis OSS поддерживает динамическое обновление конфигураций расписания Pod при изменении заболеваний узлов. Вы можете добавлять, изменять или удалять терпимости Pod, не прерывая текущую службу Redis.

## Соображения

- **Ограничения на перемещение Pod**: Миграция Pod не поддерживается во время обновления конфигурации расписания. При изменении терпимостей Pod убедитесь, что выбранные узлы после обновления включают все узлы, которые были частью развертывания до обновления.

- **Требования к доступности узлов**: Если вы настроили метки узлов для планирования, количество узлов, соответствующих как терпимостям Pod, так и заболеваниям узлов, должно быть больше или равно количеству реплик. Кроме того, обновленный выбор узлов должен либо соответствовать ранее доступным узлам, либо включать новые узлы в дополнение к исходному набору узлов для успешного обновления конфигурации расписания.

## Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. В левом навигационном меню щелкните **Redis**.

    2. Щелкните на ***название пространства имен***.

    3. Щелкните на ***название экземпляра***.

    4. Щелкните кнопку **Действия** в правом верхнем углу > **Обновить конфигурацию расписания**.

    5. Измените конфигурацию расписания по мере необходимости:
       - Обновите метки узлов для уточнения выбора подходящих узлов
       - Добавьте или измените терпимости для учета изменений в заболеваниях узлов
       - Настройте правила аффинности для оптимизации размещения экземпляра

    6. Щелкните **Обновить**, чтобы применить изменения.
  </Tab>
</Tabs>

После обновления конфигурации расписания система применит изменения, сохраняя непрерывность обслуживания. Оператор Redis будет уважать новые параметры расписания при создании или замене Pod во время последующих операций.
