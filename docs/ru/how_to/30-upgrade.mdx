# Обновление патч-версии

Обновления патч-версии Redis обеспечивают исправления безопасности и улучшения стабильности. Процесс обновления использует стратегию поэтапного обновления, последовательно перезапуская поды, чтобы поддерживать доступность сервиса в течение всей операции. Рекомендуется стратегическое планирование обновлений в рамках окон обслуживания, чтобы минимизировать потенциальное воздействие на рабочие нагрузки в производственной среде.

При появлении доступных патч-версий рекомендуется быстро их принимать, чтобы обеспечить вашим экземплярам Redis доступ к последним исправлениям безопасности и улучшениям функций.

## Настройка стратегии обновления

Вы можете определить, как патч-обновления применяются к вашим экземплярам Redis, настроив политику обновления.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Перейдите на страницу сведений об экземпляре.

    2. Выберите **Политика обновления патч-версии** из меню действий.

    3. Выберите соответствующую стратегию обновления:
       - **Ручное**: Обновления требуют явного одобрения администратора
       - **Автоматическое**: Обновления автоматически применяются в рамках окон обслуживания

    4. Нажмите **Обновить**, чтобы применить конфигурацию.
  </Tab>
</Tabs>

## Выполнение обновления отдельного экземпляра

Обновления отдельных экземпляров обеспечивают детальный контроль над процессом обновления, позволяя целенаправленную проверку перед развертыванием изменений в вашей среде.

### Предварительные условия

- Доступное обновление патч-версии обозначено значком обновления в списке экземпляров
- Статус экземпляра должен быть `Работает` перед началом обновления

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Получите доступ к странице сведений об экземпляре.

    2. Нажмите **Обновить сейчас** рядом с уведомлением об обновлении версии.

    3. Просмотрите полный журнал изменений, подробно описывающий изменения в версиях компонентов и плагинов.

    4. Нажмите **Обновить сейчас**, чтобы начать процесс.

    5. Отслеживайте статус экземпляра, пока он не вернется в состояние `Работает`, подтверждая успешное завершение обновления.
  </Tab>
</Tabs>

## Выполнение операции массового обновления

Функция массового обновления позволяет эффективно управлять обновлениями нескольких экземпляров Redis, упрощая процесс обслуживания для больших развертываний.

### Предварительные условия

- Привилегии администратора платформы
- Целевые экземпляры должны находиться в состоянии `Работает`

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Перейдите в раздел **Управление обновлением** в левой навигационной панели.

    2. Выберите соответствующий **оператор**, чтобы получить доступ к его интерфейсу управления.

    3. Выберите целевые экземпляры для обновления и нажмите **Массовое обновление**.

    4. Просмотрите подробную информацию о изменениях версий для всех затронутых компонентов.

    5. Нажмите **Обновить сейчас**, чтобы начать координированный процесс обновления.

    6. Отслеживайте ход обновления, пока все экземпляры не вернутся в состояние `Работает`.
  </Tab>
</Tabs>
