# 补丁版本升级

Redis 补丁版本升级提供安全修复和稳定性改进。升级过程采用滚动更新策略，逐一重启 Pods，以确保在操作过程中服务的可用性。建议在维护窗口期间合理安排升级，以最小化对生产工作负载的潜在影响。

当补丁版本可用时，建议及时进行升级，以确保您的 Redis 实例获得最新的安全补丁和功能增强。

## 设置升级策略

您可以通过配置升级策略来定义补丁升级如何应用于您的 Redis 实例。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 进入实例详情页面。

    2. 从操作菜单中选择 **补丁版本升级策略**。

    3. 选择适当的升级策略：
       - **手动**：升级需要明确的管理员批准
       - **自动**：升级将在维护窗口期间自动应用

    4. 点击 **更新** 以应用配置。
  </Tab>
</Tabs>

## 执行单个实例升级

单个实例升级提供了对升级过程的细粒度控制，允许在将更改推广到环境中之前进行有针对性的验证。

### 先决条件

- 可用的补丁版本升级在实例列表中通过升级图标指示
- 实例状态必须为 `运行中`，才能启动升级

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 访问实例详情页面。

    2. 在版本升级通知旁点击 **立即升级**。

    3. 查看详尽的变更日志，了解组件和插件版本的更改。

    4. 点击 **立即升级** 以启动升级过程。

    5. 监控实例状态，直到其返回为 `运行中`，以确认升级成功完成。
  </Tab>
</Tabs>

## 执行批量升级操作

批量升级功能使得在多个 Redis 实例之间高效管理升级成为可能，从而简化大规模部署的维护过程。

### 先决条件

- 平台管理员权限
- 目标实例必须处于 `运行中` 状态

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏导航到 **升级管理**。

    2. 选择适当的 **操作员** 以访问其管理界面。

    3. 选择要升级的目标实例并点击 **批量升级**。

    4. 查看所有受影响组件的详细版本更改信息。

    5. 点击 **立即升级** 以开始协调升级过程。

    6. 监控升级进度，直到所有实例返回 `运行中` 状态。
  </Tab>
</Tabs>
