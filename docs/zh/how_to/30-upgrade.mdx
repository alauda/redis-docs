# 小版本升级

随着漏洞修复或新功能发布等需求，中间件会出现小版本升级。在小版本升级过程中，将逐个重启容器组，可能会造成业务的短暂中断。因此，建议选择合适的时间窗口进行升级。在出现可升级的小版本时，建议尽快升级，以应用组件的最新修复。

## 设置小版本升级策略

### 操作步骤

<Tabs>
<Tab label="Web Console">
1. 进入实例详情页。

2. 单击 **小版本升级策略**。

3. 选择 **手动** 或 **自动**。

4. 单击 **更新**，完成设置。
</Tab>
</Tabs>

## 小版本升级

### 前提条件

出现可用小版本时，实例列表页会展示可升级图标，实例详情页会出现升级提示。

### 操作步骤

<Tabs>
<Tab label="Web Console">
1. 进入实例详情页。

2. 单击升级提示旁边的 **立即升级**。

3. 阅读升级提示，了解组件和插件的版本变更。

4. 单击 **立即升级**。

5. 等待实例状态变为运行中，升级成功。
</Tab>
</Tabs>

## 小版本批量升级

### 前提条件

用户需具备平台管理员权限。

### 操作步骤

<Tabs>
<Tab label="Web Console">
1. 在左侧导航栏中，单击 **Operators**。

2. 选择需要操作的 **operator**，进入 operator 详情页。

3. 选中需要升级的实例，单击 **批量升级**。

4. 阅读升级提示，了解组件和插件的版本变更。

5. 单击 **立即升级**。

6. 等待实例状态变为运行中，升级成功。
</Tab>
</Tabs>
