# 安装

本指南概述了 Alauda Cache Service for Redis OSS 的安装过程，包括其所需的依赖项。请按顺序执行以下步骤，以确保正确部署。

## 安装 Alauda Container Platform Data Services Essentials

::: tip 可选
Alauda Container Platform Data Services Essentials 提供了 Web 控制台管理界面组件。对于仅使用 CLI 管理的环境，可以跳过此安装。
:::

### 先决条件

1. 获取与您的平台版本兼容的 Alauda Container Platform Data Services Essentials 安装包。
2. 使用平台的应用发布功能将 Alauda Container Platform Data Services Essentials 发布到 `global` 集群。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在平台的左侧导航面板中，导航到 **Marketplace** > **Cluster Plugins**。
    2. 从集群选择器中选择 `global` 集群。
    3. 在插件列表中找到 **Alauda Container Platform Data Services Essentials**，并点击 **安装**。
  </Tab>
</Tabs>

## 安装 Alauda Container Platform Data Services RDS Framework

::: tip 可选
Alauda Container Platform Data Services RDS Framework 提供了 API 服务管理层。对于仅使用 CLI 管理的环境，可以跳过此安装。
:::

### 先决条件

1. 获取与您的平台版本兼容的 Alauda Container Platform Data Services RDS Framework 安装包。
2. 使用平台的应用发布功能将 Alauda Container Platform Data Services RDS Framework 发布到希望具备 Alauda 应用服务 Web 控制台功能的任意集群。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 访问 **平台管理** 界面。
    2. 在左侧导航面板中，导航到 **Marketplace** > **OperatorHub**。
    3. 找到 **Alauda Container Platform Data Services RDS Framework**，点击 **安装** 以访问部署配置页面。

    按如下方式配置安装参数：

    | 参数                      | 推荐配置                                                                                                                  |
    | :----------------------- | :---------------------------------------------------------------------------------------------------------------------- |
    | **渠道**                 | **稳定版** - 生产就绪的发行渠道                                                                                        |
    | **版本**                 | 最新可用版本（默认选中）                                                                                              |
    | **安装模式**             | **集群** - 提供一个单独的 Operator 实例，管理跨所有命名空间的 Redis 资源，优化资源利用率                               |
    | **安装位置**             | **推荐命名空间** - 如果适当的命名空间不存在，系统将自动创建                                                          |
    | **升级策略**             | **手动** - Operator 升级需要明确批准，提供受控的更新管理                                                              |

    4. 在安装对话框中审核配置设置，然后点击 **安装** 部署 RDS 框架 Operator。
  </Tab>
</Tabs>

## 安装 Alauda Cache Service for Redis OSS

### 先决条件

1. 获取与您的平台版本兼容的安装包。
2. 将插件上传到任何计划利用 Alauda Cache Service for Redis OSS 业务组件功能的集群。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 访问 **平台管理** 界面。
    2. 在左侧导航面板中，导航到 **Marketplace** > **OperatorHub**。
    3. 找到 **Alauda Cache Service for Redis OSS**，点击 **安装** 以访问部署配置页面。

    按如下方式配置安装参数：

    | 参数                      | 推荐配置                                                                                                                  |
    | :----------------------- | :---------------------------------------------------------------------------------------------------------------------- |
    | **渠道**                 | 根据您的平台版本自动选择合适的渠道                                                                                    |
    | **版本**                 | 最新可用版本（默认选中）                                                                                              |
    | **安装模式**             | **集群** - 提供一个单独的 Operator 实例，管理跨所有命名空间的 Redis 资源，优化资源利用率                               |
    | **安装位置**             | **推荐命名空间** - 如果适当的命名空间不存在，系统将自动创建                                                          |
    | **升级策略**             | **手动** - Operator 升级需要明确批准，提供受控的更新管理                                                              |

    4. 在安装对话框中审核配置设置，然后点击 **安装** 部署 Alauda Cache Service for Redis OSS。
  </Tab>
</Tabs>
