# 安装

本指南概述了 Alauda Cache Service for Redis OSS 的安装过程，包括其所需依赖项。请按以下步骤顺序操作，以确保正确部署。

## 安装 Alauda Application Services Essentials

::: tip 可选
Alauda Application Services Essentials 提供 Web Console 管理界面组件。如果您的环境仅使用 CLI，您可以跳过此步骤。
:::

### 前提条件

1. 获取与您的平台版本兼容的 Alauda Application Services Essentials 安装包。
2. 使用平台的应用发布功能将 Alauda Application Services Essentials 发布到 `global` 集群。

### 操作步骤

<Tabs>
  <Tab label="Web Console">
    1. 在平台的左侧导航面板中，导航到 **市场** > **集群插件**。
    2. 从集群选择器中选择 `global` 集群。
    3. 在插件列表中找到 **Alauda Application Services Essentials** 并点击 **安装**。
  </Tab>
</Tabs>

## 安装 Alauda Application Services RDS Framework

::: tip 可选
Alauda Application Services RDS Framework 提供 API 服务管理层。如果您的环境仅使用 CLI，您可以跳过此步骤。
:::

### 前提条件

1. 获取与您的平台版本兼容的 Alauda Application Services RDS Framework 安装包。
2. 使用平台的应用发布功能将 Alauda Application Services RDS Framework 发布到任何需要 Alauda Application Services Web Console 能力的集群。

### 操作步骤

<Tabs>
  <Tab label="Web Console">
    1. 访问 **平台管理** 界面。
    2. 在左侧导航栏中，导航到 **市场** > **操作员中心**。
    3. 找到 **Alauda Application Services RDS Framework** 并点击 **安装** 以访问部署配置页面。

    配置安装参数如下：

    | 参数                     | 推荐配置                                                                                                                |
    | :---------------------- | :------------------------------------------------------------------------------------------------------------------- |
    | **频道**                 | **stable** - 生产就绪的分发频道                                                                                     |
    | **版本**                 | 最新可用版本（默认选择）                                                                                            |
    | **安装模式**             | **集群** - 提供单个 Operator 实例，管理所有命名空间中的 Redis 资源，优化资源利用率                                   |
    | **安装位置**             | **推荐命名空间** - 如果不存在，系统将自动创建适当的命名空间                                                        |
    | **升级策略**             | **手动** - Operator 升级需要明确批准，以提供受控的更新管理                                                          |

    4. 在安装对话框中查看配置设置，点击 **安装** 部署 RDS Framework Operator。
  </Tab>
</Tabs>

## 安装 Alauda Cache Service for Redis OSS

### 前提条件

1. 获取与您的平台版本兼容的安装包。
2. 上传插件到任何计划利用 Alauda Cache Service for Redis OSS 商业组件能力的集群。

### 操作步骤

<Tabs>
  <Tab label="Web Console">
    1. 访问 **平台管理** 界面。
    2. 在左侧导航栏中，导航到 **市场** > **操作员中心**。
    3. 找到 **Alauda Cache Service for Redis OSS** 并点击 **安装** 以访问部署配置页面。

    配置安装参数如下：

    | 参数                     | 推荐配置                                                                                                                |
    | :---------------------- | :------------------------------------------------------------------------------------------------------------------- |
    | **频道**                 | 根据您的平台版本自动选择适当的频道                                                                                   |
    | **版本**                 | 最新可用版本（默认选择）                                                                                            |
    | **安装模式**             | **集群** - 提供单个 Operator 实例，管理所有命名空间中的 Redis 资源，优化资源利用率                                   |
    | **安装位置**             | **推荐命名空间** - 如果不存在，系统将自动创建适当的命名空间                                                        |
    | **升级策略**             | **手动** - Operator 升级需要明确批准，以提供受控的更新管理                                                          |

    4. 在安装对话框中查看配置设置，点击 **安装** 部署 Alauda Cache Service for Redis OSS。
  </Tab>
</Tabs>
