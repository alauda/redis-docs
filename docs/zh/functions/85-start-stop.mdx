# 启停实例

Alauda Redis OSS 缓存服务提供一键启停功能，以支持在管理中间件实例时的操作灵活性。这一功能对于在维护窗口、高节点负载场景或开发/测试环境中临时释放资源而不删除实例尤为重要。高效管理实例状态有助于优化资源利用率并降低运营成本。

## 停止实例

优雅地停止 Redis 实例会暂停其操作，同时保留配置和持久数据。

### 前提条件

实例必须处于以下状态之一：**处理中**、**运行中**、**异常**或**未知**。

### 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    > **建议**：在停止实例之前执行完整备份，以防止潜在的数据丢失。

    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 单击 **操作** > **停止**，并确认操作。
  </Tab>
</Tabs>

### 停止后无法使用的功能

| 功能                     | 说明                                                                                           |
| ----------------------- | ---------------------------------------------------------------------------------------------- |
| **终端控制台**             | 无法访问 Redis 实例的终端。                                                                  |
| **操作**                 | 实例操作，包括重启、更新和调度配置更改不可用。                                               |
| **拓扑**                 | 由于实例组件未运行，拓扑可视化不可用。                                                       |
| **参数配置**              | 无法进行参数更新和配置导出。                                                                  |
| **用户管理**              | 用户管理功能无法访问。                                                                        |
| **备份与恢复**            | 不能启动新的备份操作。                                                                        |
| **巡检**                 | 停止实例的巡检指标和结果无效。                                                                |
| **访问方式**              | 路由信息可能不可用或无法访问。                                                                |

## 启动实例

启动之前停止的 Redis 实例可恢复其完整功能和服务可用性。

### 前提条件

实例必须处于**停止中**或**已停止**状态。

### 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 单击 **操作** > **启动**，并确认操作。
  </Tab>
</Tabs>

启动操作成功后，实例将转换为 **运行中** 状态，所有功能将再次可用。
