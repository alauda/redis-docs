# 备份与恢复

为防止数据丢失，建议您及时进行备份。一旦出现数据库异常，可以恢复已备份的数据。

## 前提条件

在进行备份之前，请根据数据量声明存储，具体为持久卷声明（PVC）。持久卷声明封装了存储资源请求的配置，并将根据请求的访问模式、存储大小以及其他信息，自动匹配集群中合适的持久卷。您可以在容器平台中配置持久卷声明。

## 立即备份

对正在运行的 Redis 实例进行立即备份。

### 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 在 **备份与恢复** 页签中，单击 **创建备份**。

    5. 设置参数并单击 **创建**。备份的时长取决于数据量和网络状况，请耐心等待。
  </Tab>
</Tabs>

## 自动备份

指定备份时间，过后平台将自动备份 Redis 实例，并定期进行备份。

### 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 在 **备份与恢复** 页签中，单击 **自动备份设置** 区域的编辑按钮 ![](../assets/44buttoneditblue.png)。

    5. 启用 **自动备份** 并设置参数。

    6. 单击 **更新**。
  </Tab>
</Tabs>

## 恢复

将备份数据恢复到新的 Redis 实例。

### 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 进入源实例的 **备份与恢复** 页签，单击 ![](../assets/44buttonanniu.png) > **恢复**。

    4. 配置实例参数并单击 **恢复**。请耐心等待，直到目标实例处于 **运行中** 状态。
  </Tab>
</Tabs>
