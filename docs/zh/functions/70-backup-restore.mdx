# 备份与恢复

为防止数据丢失，建议及时进行备份。一旦出现数据库异常，您可以从备份中恢复数据。

灵雀云 Redis OSS 缓存服务支持两种备份类型：S3（Amazon Simple Storage Service）备份和 PVC（PersistentVolumeClaim）备份。

- **S3 备份：** 将数据备份到 S3 存储。
- **PVC 备份：** 使用持久卷声明（PVC）将数据备份到持久卷（PV）。该方法要求持久卷支持 Kubernetes 节点迁移和 ReadWriteMany 访问模式。适合的持久卷类型示例包括 NFS 和 Ceph。

## 前提条件

要将数据备份到您自己的 S3 存储桶，请向管理员确认 S3 存储已为您的项目正确配置。否则，请确保您的业务集群提供可以配置网络存储的 `StorageClass`。

## 立即备份

对正在运行的 Redis 实例进行立即备份。

### 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 在 **备份与恢复** 页签中，单击 **创建备份**。

    5. 设置参数并单击 **创建**。备份的持续时间取决于数据量和网络状况，请耐心等待。
  </Tab>
</Tabs>

## 自动备份

指定备份时间，此后平台将自动备份 Redis 实例并定期进行备份。

### 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 在 **备份与恢复** 页签中，单击 **自动备份设置** 区域的编辑按钮 ![](../assets/44buttoneditblue.png)。

    5. 启用 **自动备份** 并设置参数。

    6. 单击 **更新**。
  </Tab>
</Tabs>

## 恢复

将备份数据恢复到新的 Redis 实例。

### 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 进入源实例的 **备份与恢复** 页签，单击 ![](../assets/44buttonanniu.png) > **恢复**。

    4. 配置实例参数并单击 **恢复**。请耐心等待，直到目标实例处于 **运行中** 状态。
  </Tab>
</Tabs>
