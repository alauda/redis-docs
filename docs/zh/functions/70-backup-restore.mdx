# 备份恢复

为防止数据丢失，建议您及时进行备份。一旦出现数据库异常，可恢复已备份数据。


## 前提：声明存储

进行备份前，请根据数据量声明存储，即持久卷声明（PVC，Kubernetes PersistentVolumeClaim）。持久卷声明封装了存储资源请求配置，会根据请求的访问模式、存储大小等信息，自动匹配集群中合适的持久卷。您可在 Container Platform 中配置持久卷声明。

## 立即备份（全量）

立即备份正在运行中的 Redis 实例。

### 操作步骤

<Tabs>
<Tab label="Web Console">

1. 在左侧导航栏中，单击 **Redis**。

2. 单击 ***命名空间的名称***。

2. 单击 ***实例名称*** 。
    
3. 在 **备份恢复** 页签中，单击 **创建数据备份**。

4. 设置参数，并单击 **创建**。备份耗时与数据量及网络状况有关，请耐心等待。

</Tab>
</Tabs>


## 自动备份（全量）



指定备份时间，到期后平台将自动备份 Redis 实例，且将周期性执行备份。

### 操作步骤
#
<Tabs>
<Tab label="Web Console">

1. 在左侧导航栏中，单击 **Redis**。

2. 单击 ***命名空间的名称***。

2. 单击 ***实例名称*** 。
    
3. 在 **备份恢复** 页签中，单击 **自动备份设置** 区域的![](../assets/44buttoneditblue.png)

4. 开启 **自动备份**，并设置参数。

5. 单击 **更新**。

</Tab>
</Tabs>

## 恢复

将备份数据恢复至新的 Redis 实例。

### 操作步骤

<Tabs>
<Tab label="Web Console">

1. 在左侧导航栏中，单击 **Redis**。

2. 单击 ***命名空间的名称***。

3. 进入源实例的 **备份恢复** 页签，单击 ![](../assets/44buttonanniu.png) > **恢复**。

4. 配置实例参数，单击 **恢复**。请耐心等待，直至目标实例为 **运行中** 状态。
    
</Tab>
</Tabs>
