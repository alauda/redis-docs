# 调度配置

灵雀云的Redis OSS缓存服务支持在节点污点变化时动态更新Pods的调度配置。您可以添加、更改或删除Pods的容忍，而无需中断正在进行的Redis服务。

## 注意事项

- **Pods迁移限制**：在更新调度配置期间不支持Pod迁移。在修改Pods容忍时，请确保更新后选择的节点包含所有在更新前参与部署的节点。

- **节点可用性要求**：如果您已为调度配置了节点标签，则与Pods容忍和节点污点相匹配的节点数量必须大于或等于副本数量。此外，更新后的节点选择必须与之前可用的节点匹配，或在原有节点集的基础上新增节点，调度配置才能成功更新。

## 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 单击右上角的 **操作**按钮 > **更新调度配置**。

    5. 根据需要修改调度配置：
       - 更新节点标签以细化合格节点的选择
       - 添加或修改容忍，以适应节点污点的变化
       - 调整亲和性规则以优化实例位置

    6. 单击 **更新**以应用更改。
  </Tab>
</Tabs>

更新调度配置后，系统将应用更改，同时保持服务的连续性。Redis Operator在后续操作过程中创建或替换Pods时将遵循新的调度参数。
