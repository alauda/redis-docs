# 调度配置

节点污点发生变化时，支持添加、更改、删除 Pods 容忍。更新调度配置不会影响正常提供服务。

## 注意事项

* Pods 不支持迁移，变更 Pods 容忍后可选择节点需包含更新前已有的节点。

* 如果您已配置节点标签，Pods 容忍和节点污点相匹配的节点数量需满足大于或等于副本数，且变更后的可用节点需和原可用节点保持一致或在原可用节点上添加新的节点，调度配置才可正常更新。

## 操作步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，单击 **Redis**。

    2. 单击 ***命名空间的名称***。

    3. 单击 ***实例的名称***。

    4. 单击右上角 **操作** > **更新调度配置**。

    5. 按需求修改配置。

    6. 单击 **更新**。
  </Tab>
</Tabs>
