# 介绍

## 什么是 Redis OSS？

**Redis OSS** 是一款开源的**内存数据结构存储系统**，支持多种数据结构（如字符串、哈希、列表、集合、有序集合等），并提供持久化、高可用、分布式集群等功能。凭借其高性能和低延迟的特性，Redis 被广泛应用于缓存、消息队列、实时数据分析等场景。其核心部署模式包括：
- **Standalone**：单节点部署，适用于开发测试环境。
- **Sentinel**：基于哨兵的高可用架构，提供自动故障转移和监控。
- **Cluster**：分布式分片集群，支持横向扩展与数据自动分区。

## 什么是 Alauda Cache Service for Redis OSS？

**Alauda Cache Service for Redis OSS** 基于 Kubernetes（K8s）自定义了多个控制器（Operator），专为简化 Redis 实例在 Kubernetes 集群中的生命周期管理而设计。它通过扩展 Kubernetes API，提供声明式配置和自动化运维能力，使用户能够以云原生的方式高效部署、管理 Redis 集群，同时保障生产环境的稳定性和可扩展性。

### 核心功能

1. **多架构支持**
   - 支持 **Standalone**、**Sentinel**、**Redis Cluster** 三种部署模式，覆盖开发、测试及生产需求。

   | 模式         | 适用场景                        |
   |--------------|-------------------------------|
   | Standalone   | 开发测试/轻量级缓存              |
   | Sentinel     | 读写分离/主从热备                |
   | Cluster      | 海量数据/水平扩展                |

2. **版本兼容性**
   - 支持 Redis 5.0、6.0、7.2。
3. **访问控制与安全**
   - 支持 TLS 加密
   - 集成 **Redis ACL**（Access Control List），支持细粒度的用户权限管理(Redis 5.0 不支持此功能)。
4. **网络与访问方式**
   - 支持 **NodePort** 和 **LoadBalancer** 类型的服务暴露，可指定 NodePort 端口。
   - 兼容 **IPv4** 和 **IPv6** 网络环境。
5. **弹性扩缩容**
   - 支持在线横向扩缩容（Cluster 模式）与纵向资源调整，无需停机。
6. **版本升级与高可用**
   - 提供**优雅的版本升级**机制，确保服务不中断。
   - 通过多副本、故障自动恢复等机制保障**生产环境高可用性**。
7. **调度策略定制**
   - 支持 **NodeSelector**、**Toleration** 和 **Affinity**，灵活控制 Pod 调度逻辑。
8. **自动化运维能力**
   - ​**滚动扩缩容**：基于配置滚动调整分片或副本数量，自动处理数据重分布（Cluster）或主从关系（Sentinel）
   - ​**无中断升级**：逐个节点替换并同步数据，保障服务连续性
   - ​**配置热更新**：修改 ConfigMap 后自动注入新配置，无需重启实例

## 为什么选择 Alauda Cache Service for Redis OSS

Alauda Cache Service for Redis OSS 将 Redis 的复杂运维逻辑（如集群拓扑管理、版本升级、扩缩容）抽象为 Kubernetes 原生资源，使运维人员能够通过简单的 YAML 声明式配置完成操作。它显著降低了在 Kubernetes 中管理 Redis 的复杂度，同时兼顾灵活性、安全性及生产级稳定性，是云原生场景下 Redis 集群的理想管理工具。
